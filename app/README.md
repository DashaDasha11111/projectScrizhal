# Скрижаль

Мобильное Android-приложение для управления церковными делами Красноярской епархии. Автоматизирует учёт наград духовенства, распределение служб на праздничные дни и взаимодействие с производственными мастерскими.

---

## Возможности

### Роли пользователей

Приложение поддерживает три роли, каждая со своим интерфейсом и набором функций:

#### Митрополит (администратор)
- **Панель событий** — список ближайших событий всего духовенства (дни рождения, именины, годовщины рукоположения, ожидаемые награды). Фильтрация по типу события, наименованию и уровню награды (епархиальная / московская), а также по периоду (неделя / месяц / позже).
- **Реестр клириков** — справочник духовенства с поиском, фильтрацией по типу (белое духовенство / монашествующие) и сану. Добавление, редактирование и удаление записей.
- **Реестр церквей** — справочник храмов Красноярского края с поиском. Карточка каждого храма содержит адрес, настоятеля и количество клириков.
- **Заказы** — управление заказами наград в мастерские (статусы: Отправлен / В работе / Выполнен). Формирование и экспорт прошений в Московскую Патриархию в виде HTML-документов.
- **Литургии** — назначение духовенства на праздничные богослужения в храмах. Фильтрация по наличию назначений.

#### Клирик
- **Профиль** — личные данные и история наград.
- **Литургии** — предстоящие праздничные служения с назначением.
- **Уведомления** — внутренние уведомления о новых назначениях.

#### Мастерская (производство)
- **Новые заказы** — входящие заказы от Митрополита. Принятие в работу или отклонение.
- **В работе** — заказы в производстве. Отметка о выполнении.
- **Архив** — выполненные заказы с поиском и фильтрацией.

## Система наград

Ядро приложения — движок канонических правил Русской Православной Церкви (п. 2.3.3–2.3.12):

**Белое духовенство (10 ступеней):**
Набедренник → Камилавка → Наперсный крест → Палица → Крест с украшениями → Протоиерей → Литургия (Иже Херувимы) → Литургия (Отче наш) → Митра → Второй крест

**Монашествующие (8 ступеней):**
Аналогично, без Камилавки и Протоиерея; Митра заменяется на Архимандрит

Для каждой ступени задан минимальный срок после предыдущей награды и минимальный общий стаж от даты рукоположения. На основе истории наград клирика приложение:
- автоматически рассчитывает дату, с которой клирик может быть представлен к следующей награде;
- строит полный прогноз наград;
- определяет, оформляется ли награда через епархиальную мастерскую или требует прошения в Москву.

## Стек технологий

| Компонент | Технология |
|---|---|
| Язык | Kotlin |
| Минимальная версия Android | 8.0 (API 26) |
| Целевая версия Android | API 36 |

| Система сборки | Gradle Kotlin DSL |

## Сборка проекта

### Необходимые инструменты

- [Android Studio](https://developer.android.com/studio) Hedgehog или новее
- JDK 17+
- Android SDK (API 26–36)

### Шаги

1. Клонируй репозиторий:
   ```bash
   git clone <ссылка на репозиторий>
   cd Scrizhal3
   ```

2. Положи файл `google-services.json` (от Firebase-проекта) в папку `app/`.

3. Открой проект в Android Studio: **File → Open → выбери папку `Scrizhal3`**.

4. Дождись синхронизации Gradle.

5. Собери APK: **Build → Build Bundle(s) / APK(s) → Build APK(s)**.

Готовый APK будет в: `app/build/outputs/apk/debug/app-debug.apk`

### Сборка через командную строку

```bash
./gradlew assembleDebug
```

APK окажется там же: `app/build/outputs/apk/debug/app-debug.apk`
